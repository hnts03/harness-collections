## 실행 전 확인

1. 유사한 에이전트가 이미 존재하는지 확인한다:
   ```bash
   ls .claude/agents/ 2>/dev/null
   ```
   유사 에이전트가 존재하면 새로 생성하지 않고 기존 에이전트 확장을 제안한다.

2. 에이전트 명세가 준비되어 있는지 확인한다. 없으면 다음을 유저에게 묻는다:
   - 에이전트 이름
   - 에이전트 역할 (무엇을 하는가)
   - 사용할 모델 (opus/sonnet)
   - 입력/출력 프로토콜

---

## 에이전트 정의 파일 생성 규칙

생성 경로: `.claude/agents/<name>.md`

### frontmatter 필수 항목

```yaml
---
name: <name>
description: <에이전트의 역할을 명확히 기술하는 한 줄 설명>
model: opus  # 또는 sonnet
---
```

**모델 선택 기준:**

| 모델 | 역할 유형 |
|------|---------|
| `opus` | 복잡한 판단, 설계, 분석, 리뷰, QA |
| `sonnet` | 명확히 정의된 task 실행, 구조화된 문서 작성 |

판단 복잡도가 높은 역할은 opus를 선택한다.

### 본문 필수 섹션

1. **역할 및 전문성 선언** — 이 에이전트가 누구인가, 어떤 전문성을 가지는가
2. **컨텍스트 파악** — 어떤 Context 블록을 받아 어떻게 읽는가
3. **실행 단계 (STEP N)** — 번호 순서로 명확히 나열
4. **완료 보고** — 반환 형식 (STATUS: completed/failed 포함)

**팀 통신 프로토콜 (에이전트 팀 모드인 경우):**
- 메시지 수신 대상
- 메시지 발신 대상
- 작업 요청 범위

**실패 원칙:** 재시도하지 않고 실패 결과를 반환한다 (무한 루프 방지).

## 생성 후 검증

```bash
# 파일 위치 확인
ls .claude/agents/<name>.md

# frontmatter 확인
head -5 .claude/agents/<name>.md
```

생성된 에이전트 정의를 유저에게 제시하고 피드백을 받는다.
